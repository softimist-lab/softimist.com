In the modern media landscape, content comes from everywhere — studios, independent creators, news agencies, sports leagues, user-generated platforms. Managing this flood of content across multiple distribution channels is one of the hardest engineering challenges in media technology. This is the problem **PlayMist Aggregator** was built to solve.

## The Content Chaos Problem

Most media companies today deal with:

- **Dozens of content sources** — each with different formats, APIs, and metadata schemas
- **Complex rights windows** — content availability varies by region, time, and platform
- **Metadata inconsistency** — one source calls it "genre", another calls it "category"
- **Manual workflows** — copy-pasting between spreadsheets and CMS platforms
- **Distribution complexity** — each platform (Netflix, YouTube, local OTT) has unique requirements

The result? Teams spend more time wrangling data than creating value.

### Metadata Normalization

Raw metadata from different sources is messy. The Aggregator normalizes everything into a consistent schema:

| Source Field | Normalized Field | Type |
|-------------|-----------------|------|
| `title`, `name`, `headline` | `content.title` | string |
| `desc`, `description`, `synopsis` | `content.description` | string |
| `genre`, `category`, `type` | `content.genres[]` | string[] |
| `dur`, `duration`, `length` | `content.duration` | number (seconds) |
| `rating`, `mpaa`, `certification` | `content.ratings[]` | Rating[] |

## Real-Time Sync & Conflict Resolution

When the same content exists in multiple sources, conflicts are inevitable. The Aggregator uses a priority-based resolution system:

1. **Source priority** — Higher-priority sources win on conflicting fields
2. **Recency** — More recently updated data takes precedence
3. **Completeness** — The richer metadata record wins
4. **Manual override** — Editorial team can always override automated decisions

All changes are tracked in an audit log, so you can always see what changed, when, and why.
