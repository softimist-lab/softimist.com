In the modern media landscape, content comes from everywhere ‚Äî studios, independent creators, news agencies, sports leagues, user-generated platforms. Managing this flood of content across multiple distribution channels is one of the hardest engineering challenges in media technology. This is the problem **PlayMist Aggregator** was built to solve.

## The Content Chaos Problem

Most media companies today deal with:

- **Dozens of content sources** ‚Äî each with different formats, APIs, and metadata schemas
- **Complex rights windows** ‚Äî content availability varies by region, time, and platform
- **Metadata inconsistency** ‚Äî one source calls it "genre", another calls it "category"
- **Manual workflows** ‚Äî copy-pasting between spreadsheets and CMS platforms
- **Distribution complexity** ‚Äî each platform (Netflix, YouTube, local OTT) has unique requirements

The result? Teams spend more time wrangling data than creating value.

### Source-Agnostic Ingestion

Connect any content source with our pre-built adapters or create custom ones:

```typescript
// Connecting a new content source
const source = await aggregator.sources.create({
  name: 'BBC Content Feed',
  type: 'mrss', // Media RSS
  endpoint: 'https://feeds.bbc.com/content/v2',
  auth: {
    type: 'api_key',
    key: process.env.BBC_API_KEY,
  },
  schedule: {
    frequency: 'every_15_minutes',
    timezone: 'UTC',
  },
  mapping: {
    title: '$.item.title',
    description: '$.item.description',
    thumbnail: '$.item.media:thumbnail.@url',
    duration: '$.item.media:content.@duration',
    genre: '$.item.category',
  },
})
```

### Metadata Normalization

Raw metadata from different sources is messy. The Aggregator normalizes everything into a consistent schema:

| Source Field | Normalized Field | Type |
|-------------|-----------------|------|
| `title`, `name`, `headline` | `content.title` | string |
| `desc`, `description`, `synopsis` | `content.description` | string |
| `genre`, `category`, `type` | `content.genres[]` | string[] |
| `dur`, `duration`, `length` | `content.duration` | number (seconds) |
| `rating`, `mpaa`, `certification` | `content.ratings[]` | Rating[] |

### AI-Powered Enrichment

Beyond basic normalization, the Aggregator enriches your content automatically:

- **Auto-tagging** ‚Äî AI analyzes thumbnails and descriptions to generate relevant tags
- **Language detection** ‚Äî Identifies content language from audio tracks and metadata

## Rights Management Engine

Content rights are perhaps the most complex part of media operations. The Aggregator handles:

```typescript
// Define a rights window
const rights = await aggregator.rights.create({
  contentId: 'movie-2024-001',
  windows: [
    {
      platform: 'playmist-bd',
      territory: ['BD'],
      startDate: '2026-03-01',
      endDate: '2027-03-01',
      monetization: ['svod', 'tvod'],
      exclusive: true,
    },
    {
      platform: 'youtube',
      territory: ['GLOBAL'],
      startDate: '2027-03-01',
      endDate: null, // perpetual
      monetization: ['avod'],
      exclusive: false,
    },
  ],
})
```

The engine automatically:
- Prevents content from publishing outside its rights window
- Alerts teams before rights expire
- Handles territory-based restrictions
- Manages exclusivity conflicts across platforms

## Real-Time Sync & Conflict Resolution

When the same content exists in multiple sources, conflicts are inevitable. The Aggregator uses a priority-based resolution system:

1. **Source priority** ‚Äî Higher-priority sources win on conflicting fields
2. **Recency** ‚Äî More recently updated data takes precedence
3. **Completeness** ‚Äî The richer metadata record wins
4. **Manual override** ‚Äî Editorial team can always override automated decisions

All changes are tracked in an audit log, so you can always see what changed, when, and why.
